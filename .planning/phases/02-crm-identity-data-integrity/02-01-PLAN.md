<objective>
Fix LID/JID normalization at ingestion point. When a Baileys inbound message arrives from a `@lid` JID, the Contact record must store `phoneNumber = null` and `rawJid = event.remoteJid` — never write LID digits into `phoneNumber`. A BullMQ reconciliation job fires on each `connection.update: open` event to attempt resolution of contacts with null phoneNumber.

Purpose: CRM-01 — operators must never see internal LID codes stored as phone numbers. This plan establishes the schema truth and the background resolution path.
Output: Modified schema.prisma (rawJid column + nullable phoneNumber), new queue factory, service.ts updated with @lid-aware upsert branch and CONNECTED hook.
</objective>

3c. Register a Worker for LID_RECONCILIATION. Follow the Worker pattern from other services. The worker processor function:
```typescript
async function processLidReconciliation(
  job: Job<{ tenantId: string; instanceId: string }>,
  prisma: TenantPrismaClient,
  logger: pino.Logger
): Promise<void> {
  const { tenantId, instanceId } = job.data;
  const startMs = Date.now();

  // Find all contacts for this instance with null phoneNumber but non-null rawJid
  const unresolved = await prisma.contact.findMany({
    where: { instanceId, phoneNumber: null, rawJid: { not: null } },
    select: { id: true, rawJid: true, fields: true }
  });

  logger.info(
    { instanceId, count: unresolved.length, elapsedMs: Date.now() - startMs },
    "LID reconciliation: found unresolved contacts"
  );

  let resolved = 0;
  for (const contact of unresolved) {
    // sharedPhoneJid may have been stored by a prior phone-number-share event
    const sharedPhoneJid = (contact.fields as Record<string, unknown>)?.sharedPhoneJid as string | undefined;
    if (!sharedPhoneJid) continue; // Cannot resolve without sharedPhoneJid — wait for next connection

    try {
      // Call existing persistLidPhoneMapping — DO NOT duplicate its merge logic
      await persistLidPhoneMapping(prisma, instanceId, contact.rawJid!, sharedPhoneJid);
      resolved++;
    } catch (err) {
      logger.warn({ instanceId, rawJid: contact.rawJid, err }, "LID reconciliation: failed for one contact");
    }
  }

  logger.info(
    { instanceId, resolved, total: unresolved.length, elapsedMs: Date.now() - startMs },
    "LID reconciliation: complete"
  );
}
```
