<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator UI → CRM API | Query params (tags, page, pageSize) cross from panel to API |
| CRM API → Prisma (tenant DB) | Contact and memory queries executed in tenant schema context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Injection | tags query param in `hasSome` | mitigate | Tags are passed as a string array to Prisma `hasSome` — Prisma parameterizes the query; no raw SQL. Validate tags as `z.array(z.string().max(50))` in the Zod schema to prevent oversized values |
| T-02-03-02 | Injection | phone8 suffix in `contains` query | mitigate | `cleanPhone()` strips all non-digit characters before the `contains:` query — no SQL injection possible via phone input |
| T-02-03-03 | Information Disclosure | clientMemory fields in API response | accept | PersistentMemory data is AI-extracted from the contact's own conversations; operator-facing only (authenticated); no cross-tenant leakage (query scoped to tenant prisma client) |
| T-02-03-04 | Denial of Service | messages take:500 could be large | accept | 500 messages is a reasonable ceiling; each message row is ~1KB; max 500KB per contact query — acceptable. Add `take: 500` explicitly as a ceiling rather than no limit |
| T-02-03-05 | Information Disclosure | Sensitive notes in memory.notes field | accept | Operator-facing read-only display; notes are AI-captured from conversation context the operator already has access to |
</threat_model>
