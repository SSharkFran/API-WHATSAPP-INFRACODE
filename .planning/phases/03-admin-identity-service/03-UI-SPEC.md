> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 3 has `UI hint: yes` but no new pages are created. The UI surface is exclusively the existing super-admin panel (`apps/panel/app/(super-admin)/admin/`). The work is:

1. **Plan 3.3** — Ensure platform routes render correctly under real auth (no dev bypass). Fix any broken states that were masked by the bypass.
2. **No new page scaffolds.** All components already exist in `components/ui/`.

This contract governs the visual states that must be correct after Plan 3.3: auth-gated error states, loading states, and empty states on the four super-admin pages (Overview, Tenants, Billing, Settings).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline padding |
| sm | 8px | Gap between badge dot and label; compact list row padding |
| md | 16px | Default element spacing; card internal padding |
| lg | 24px | Section padding (`px-5 py-4` = 20/16 → round to 24 for section gaps) |
| xl | 32px | Layout column gaps; major card gaps |
| 2xl | 48px | Empty state vertical padding (py-16 = 64px; halved for content zone) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets on icon-only nav items: minimum 44px height — enforced by `h-7 w-7` (28px) within a larger hit area. **Action:** Any icon-only interactive added in Phase 3 must have min `h-11 w-11` (44px) to meet WCAG 2.5.5.
- Progress track height: 4px (non-interactive — exception is intentional).
- Status bar height: 1px (decorative only).

Source: `globals.css` spacing patterns, `admin/layout.tsx` nav item heights, Button component `sizeStyles`.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Space Grotesk |
| Label / Kicker | 10px (text-[10px]) | 500 (medium) | 1.4 | IBM Plex Mono, uppercase, tracking-widest |
| Heading (card) | 16px (text-base) | 600 (semibold) | 1.3 | Space Grotesk |
| Display (page title) | 20px (text-xl) | 700 (bold) | 1.2 | Space Grotesk |

Rules:
- Exactly 2 weights: **400** (body text, secondary labels) and **600/700** (headings, button labels, badge text).
- Mono font reserved for: kicker labels (`control-kicker` pattern), slug/JID values, metric numbers in badge context.
- Never use raw JID (`@s.whatsapp.net`, `@lid`) as visible text — always pass through `formatPhone()` (CRM-02, ADM requirement).

Source: `globals.css` `.text-fluid-title`, `.control-kicker`, admin page component classes, Button `sizeStyles`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`--bg-primary`) | Page background, outermost shell |
| Secondary (30%) | `#111111` (`--bg-secondary`) | Sidebar background, main content panel, card surface |
| Tertiary | `#1a1a1a` (`--bg-tertiary`) | List rows, inner cards, input backgrounds |
| Accent (10%) | `#3b82f6` (`--accent-blue`) | See reserved list below |
| Destructive | `#ef4444` (`--accent-red`) | Destructive actions only (suspend tenant, revoke key) |

**Accent reserved for:**
- Active nav item left-border (`border-l-2 border-[var(--accent-blue)]`)
- Primary action buttons (`Button variant="primary"`)
- Progress fill bar (`progress-fill`)
- Focus ring on interactive elements (`focus-ring`)
- Pulsing status dot on "Super Admin" identity indicator in sidebar
- `Badge variant="info"` dot and text tint

**Accent is NOT used for:**
- Secondary badges (use emerald/yellow/red semantic colors)
- Background fills on cards
- Text color on body copy

**Semantic colors (status badges — 10% of accent budget shared):**

| Semantic | Value | Reserved for |
|----------|-------|-------------|
| Success | `#22c55e` (`--accent-green`) | `Badge variant="success"` — ACTIVE tenant, PAID billing |
| Warning | `#eab308` (`--accent-yellow`) | `Badge variant="warning"` — PENDING billing, unverified admin |
| Error / Destructive | `#ef4444` (`--accent-red`) | `Badge variant="error"` — SUSPENDED, PAST_DUE; `Button variant="danger"` |

Source: `globals.css` accent variables, `Badge.tsx` variantStyles, `Button.tsx` variantStyles, admin/page.tsx Badge usage.

## Copywriting Contract

### Primary CTA

| Surface | CTA Label |
|---------|-----------|
| Tenant suspension confirm | "Suspender tenant" |
| Route audit (internal) | No user-facing CTA — developer tool only |

### Empty States

All empty states use the existing `<EmptyState>` component (`components/ui/EmptyState.tsx`) with a Lucide icon, a single label string, and an optional action button.

| Surface | Icon | Label | Action |
|---------|------|-------|--------|
| Tenants list — no tenants | `Users` | "Nenhum tenant cadastrado ainda." | none |
| Billing list — no records | `CreditCard` | "Nenhum registro de cobrança encontrado." | none |
| Admin route — 403 (no PLATFORM_OWNER role) | `ShieldOff` | "Acesso negado. Esta área requer permissão de Platform Owner." | none |
| Admin route — API fetch error | `AlertTriangle` | "Não foi possível carregar os dados. Tente recarregar a página." | "Recarregar" (calls `router.refresh()`) |

### Error States

| Error | Message | Recovery Path |
|-------|---------|---------------|
| 401 Unauthenticated on admin route | "Sessão expirada. Faça login novamente." | Redirect to `/login` |
| 403 Missing PLATFORM_OWNER scope | "Acesso negado. Esta área requer permissão de Platform Owner." | No action — display only |
| 500 API failure on admin data load | "Erro interno ao carregar dados. Tente novamente." | "Tentar novamente" button |
| Admin JID not resolved (ADM-03) | Surfaced only in logs — no user-facing message for this server-side event | n/a |

### Destructive Actions

| Action | Confirmation Copy | Pattern |
|--------|------------------|---------|
| Suspend tenant | "Tem certeza que deseja suspender **{nome}**? O tenant perderá acesso imediatamente." | Inline confirmation row within TenantManager — "Confirmar" (danger) + "Cancelar" (ghost) buttons, no modal |
| Reactivate suspended tenant | "Reativar **{nome}**?" | Inline confirmation — "Confirmar" (primary) + "Cancelar" (ghost) |

No modal/dialog is used for these confirmations — follow the existing inline-expansion pattern from `tenant-manager.tsx` (Phase 2 established this pattern in CRM).

Source: ROADMAP Phase 3 UAT scenarios, existing `Button.tsx` variant="danger", admin/page.tsx pattern analysis.

## Interaction States

All interactive elements must have explicit states for:

| State | Implementation |
|-------|---------------|
| Default | Defined by component base style |
| Hover | `hover:` Tailwind variant — `bg-[var(--bg-hover)]` for rows/nav; `opacity-88` for buttons |
| Focus-visible | `focus-visible:ring-2 focus-visible:ring-[var(--accent-blue)]` — already on Button and nav links |
| Disabled | `opacity-40 cursor-not-allowed` — already on Button |
| Loading | `<Skeleton>` component for data fetch; `Button loading={true}` for async actions |
| Active (nav) | `bg-[var(--bg-active)] border-l-2 border-[var(--accent-blue)]` — already in AdminSidebar |
| Error | `Badge variant="error"` for status; error copy per Copywriting Contract above |

**Reduced motion:** Existing `@media (prefers-reduced-motion: reduce)` in `globals.css` suppresses all animations. No additional work needed.

Source: `Button.tsx`, `admin/layout.tsx` nav link classes, `globals.css` animation utilities.

No new components are introduced. The following existing components are used in the admin routes under audit:

| Component | File | Usage in Phase 3 |
|-----------|------|-----------------|

| `StatCard` | `components/dashboard/stat-card.tsx` | Metric tiles on Overview page |
| `Topbar` | `components/layout/Topbar.tsx` | Breadcrumb navigation in admin shell |

**Pattern enforcement (ADM-04):** Every admin page component must call the correct API endpoint that requires `platformRole: "PLATFORM_OWNER"`. The panel itself does not enforce auth — it relies on the API returning 403. The panel must handle 403 explicitly (not show a blank/broken page).
