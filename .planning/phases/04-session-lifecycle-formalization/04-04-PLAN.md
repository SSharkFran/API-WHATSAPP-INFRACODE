<objective>
Introduce `InstanceEventBus` — a typed Node.js EventEmitter wrapper — at `apps/api/src/lib/instance-events.ts`. Wire `InstanceOrchestrator` to emit `session.activity`, `session.close_intent_detected`, and `admin.command` domain events. Wire `SessionLifecycleService` to subscribe to `session.activity` and call `recordActivity()`. This decoupling seam allows Phase 5, 6, and 7 services to subscribe to events without direct coupling to InstanceOrchestrator.

Purpose: The current architecture requires InstanceOrchestrator to directly call every downstream service — creating tight coupling that blocks incremental extraction. InstanceEventBus converts these calls to fire-and-forget events, making each subscriber independently replaceable.
Output: `instance-events.ts` (new), `service.ts` (emit calls added), `app.ts` (bus instantiated and injected).
</objective>

```typescript
// apps/api/src/lib/instance-events.ts
import { EventEmitter } from 'node:events';

export interface SessionActivityEvent {
  type: 'session.activity';
  tenantId: string;
  instanceId: string;
  remoteJid: string;
  sessionId: string;   // from Redis hash — may be empty string if session not yet persisted
}

export interface SessionCloseIntentEvent {
  type: 'session.close_intent_detected';
  tenantId: string;
  instanceId: string;
  remoteJid: string;
  sessionId: string;
  intentLabel: string; // e.g. 'ENCERRAMENTO' — static label from SESS-09 stub, LLM label in Phase 5
}

export interface AdminCommandEvent {
  type: 'admin.command';
  tenantId: string;
  instanceId: string;
  command: string;
  fromJid: string;    // admin JID
}

export type InstanceDomainEvent =
  | SessionActivityEvent
  | SessionCloseIntentEvent
  | AdminCommandEvent;

export class InstanceEventBus extends EventEmitter {
  emit(event: InstanceDomainEvent['type'], payload: InstanceDomainEvent): boolean {
    return super.emit(event, payload);
  }
  on(event: InstanceDomainEvent['type'], listener: (payload: InstanceDomainEvent) => void): this {
    return super.on(event, listener);
  }
  off(event: InstanceDomainEvent['type'], listener: (payload: InstanceDomainEvent) => void): this {
    return super.off(event, listener);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| InstanceOrchestrator → InstanceEventBus | In-process publish; all listeners run in same process |
| EventBus listener error → main message path | Listener errors must not propagate to caller or crash the process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Denial of Service | EventBus listener throws | mitigate | Every async listener wraps in `.catch(err => this.logger.error(...))` — unhandled rejections do NOT propagate to the InstanceOrchestrator emit call |
| T-04-04-02 | Denial of Service | EventBus listener backpressure | accept | Node.js EventEmitter has no built-in backpressure; listeners are lightweight (BullMQ enqueue is async, not blocking). If listener count grows beyond 10 per event type, Node will warn — acceptable for Phase 4 |
| T-04-04-03 | Tampering | admin.command event from non-admin | mitigate | `admin.command` is only emitted after `identityContext.isAdmin === true` is confirmed by AdminIdentityService — the same guard that already protects admin routes |
| T-04-04-04 | Information Disclosure | session.activity payload in event | accept | Payload contains tenantId, instanceId, remoteJid — same data already present in in-process memory at the call site; no new exposure surface |
</threat_model>
