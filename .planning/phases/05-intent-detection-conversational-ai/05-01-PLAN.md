## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→LLM prompt | Client message text is interpolated into the Groq system prompt |
| LLM response→intent label | Raw LLM string output is used to drive session state transitions |
| Groq API availability→pipeline | Groq rate limits / downtime must not block the chatbot pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering | classifyIntent() prompt construction | mitigate | Wrap client text in `<message>{text}</message>` delimiters — never interpolate rawTextInput directly into system prompt string |
| T-5-02 | Tampering | LLM response → IntentLabel | mitigate | Validate parsed label against `VALID_INTENT_LABELS` whitelist before acting; fallback to `OUTRO` if label is absent or unrecognized |
| T-5-05 | Denial of Service | GroqKeyRotator / all keys on cooldown | mitigate | Entire pre-pass is inside try/catch; catch path logs warn and returns without emitting any event — pipeline continues normally with no block |
</threat_model>
