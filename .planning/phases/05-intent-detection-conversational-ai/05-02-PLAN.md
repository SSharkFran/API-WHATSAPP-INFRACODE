<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| intent label → session state write | URGENCIA_ALTA and TRANSFERENCIA_HUMANO labels from LLM classifier drive writes to Redis/PostgreSQL session state |
| session history → admin WhatsApp message | Last 5 exchanges from session.history are sent verbatim to admin phone — may contain sensitive client data |
| client remoteJid → Redis key component | remoteJid used in Redis HSET key — same VALID_JID_PATTERN enforcement as SessionStateService |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-02 | Tampering | LLM label → setHumanTakeover | mitigate | setHumanTakeover is only called when classification.label === 'TRANSFERENCIA_HUMANO' (validated against VALID_INTENT_LABELS in Plan 5.1 before reaching this code). No raw LLM output reaches this branch. |
| T-5-03 | Elevation of Privilege | Admin notification — echo loop | mitigate | Admin notification uses sendAutomatedTextMessage() which calls rememberAutomatedOutboundEcho() internally — prevents re-processing as inbound message. NEVER use raw sock.sendMessage() for admin notifications. |
| T-5-06 | Information Disclosure | session.history in admin notification | accept | Admin phone is the tenant's own admin — they are authorized to see conversation history. Limit to last 10 messages (slice(-10)) and truncate each to 120 chars to avoid payload overflow. |
| T-5-07 | Denial of Service | urgencyScore Redis HSET on every URGENCIA_ALTA | accept | Low-frequency event; urgency is rare. HSET is O(1). Risk: negligible. |
| T-5-08 | Tampering | remoteJid in Redis urgencyScore HSET key | mitigate | Use sessionStateService.setUrgencyScore when available (which validates JID via VALID_JID_PATTERN). If direct HSET is used as fallback, the key format `session:{tenantId}:{instanceId}:{remoteJid}` is identical to SessionStateService's pattern — validate remoteJid ends in @s.whatsapp.net or @g.us before building the key. |
</threat_model>
