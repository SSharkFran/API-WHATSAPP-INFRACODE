<objective>
Implement the graceful "I Don't Know" response — replacing the current silent null path in processConversationTurn with an honest client-facing fallback message, gated admin notification (Part B only when aprendizadoContinuo is active), and hardened OrchestratorAgent fallback chain.

Purpose: IA-03 requires that the bot never leaves a client waiting in silence due to an internal failure. IA-04 requires that when the bot cannot answer, it sends an honest message to the client AND (if the aprendizadoContinuo module is active) escalates with structured context to the admin. The current codebase returns null when chatbotResult is null AND config.fallbackMessage is empty AND the module is disabled — this plan replaces that silence with the standard honest fallback.
Output: Honest fallback text in processConversationTurn, module-gated admin notification (reusing the existing ESCALATE_ADMIN admin notification pattern), passing tests for IA-03 and IA-04, and OrchestratorAgent fallback chain tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI null result → honest fallback message | A null chatbotResult (chatbot disabled, AI failure, no match) must not trigger honest fallback when session is intentionally blocked |
| client question text → admin WhatsApp message (Part B) | Client's raw question is included in the admin notification — no sanitization needed for internal admin, but must be length-bounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-04 | Spoofing | Honest fallback fires during humanTakeover | mitigate | Re-check isConversationAiBlocked immediately before sending honest fallback — if true, return silently (Pitfall 5). The re-check uses the same DB query as the entry guard (line 4073) — this is a deliberate double-check, not redundant. |
| T-5-09 | Information Disclosure | Client question in admin notification (Part B) | accept | Admin is the authorized tenant owner; client question is their business data. Limit question text to 300 chars in the notification (`.slice(0, 300)`) to prevent message size abuse. |
| T-5-10 | Tampering | aprendizadoContinuo module gate bypass | mitigate | Part B admin notification only fires when `aprendizadoContinuoModule?.isEnabled === true && verificationStatus === "VERIFIED"`. Both conditions required — same triple-equals pattern as existing ESCALATE_ADMIN path at line 4359. |
| T-5-11 | Denial of Service | Honest fallback fires on every unanswered message | accept | The honest fallback fires once per conversation turn where chatbotResult is null AND session is not blocked. In normal operation, most messages produce a chatbotResult. Rate limiting is handled by the existing session debounce mechanism (queueConversationTurn). |
</threat_model>
