<objective>
Replace fire-and-forget void calls in the message pipeline with BullMQ-backed jobs (for work that must survive restarts) or setImmediate with structured error logging (for deferred-but-disposable work), and add pino structured logging to OrchestratorAgent to replace console.log/warn/error.

Purpose: IA-05 requires that the conversation flow never stalls due to internal exceptions — fire-and-forget void calls are the primary source of silent failures that make debugging impossible and leave errors unobservable. IA-03 requires resilience — an uncaught error in a fire-and-forget call can crash the worker process, taking down all active conversations. This plan addresses the two most critical fire-and-forget patterns identified in the codebase: `triggerKnowledgeSynthesis` and `extractPersistentMemory`.
Output: BullMQ-backed knowledge synthesis jobs, setImmediate-wrapped persistent memory extraction with structured logging, pino-aware OrchestratorAgent.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BullMQ job payload → chatbotService call | Job payload { tenantId, instanceId } is used to call triggerKnowledgeSynthesis — must be validated before use |
| setImmediate callback → extractPersistentMemory | Deferred callback captures params from outer scope — must not hold stale closure references after session reset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-12 | Tampering | BullMQ job payload { tenantId, instanceId } | mitigate | triggerKnowledgeSynthesis already validates tenantId/instanceId internally via tenantPrismaRegistry.getClient(tenantId). The job payload is trusted (set by this service, never from user input). |
| T-5-13 | Denial of Service | BullMQ queue growth if knowledge synthesis jobs pile up | accept | Job has `attempts: 3` with exponential backoff and `removeOnComplete: 100, removeOnFail: 50` — queue self-limits. Synthesis is background work; queue backpressure is acceptable. |
| T-5-14 | Information Disclosure | pino logger printing { err } objects containing stack traces | accept | Internal logger only; logs are written to stdout/file, not to clients. Stack traces in error logs are expected and intentional for debugging. |
| T-5-15 | Denial of Service | setImmediate closures capturing large session objects | mitigate | Pass only scalar values (tenantId: string, instanceId: string, params.inputText: string) into setImmediate callbacks — do not capture the full params object. This prevents memory leaks when sessions are GC'd before the callback fires. Extract only the needed values before the setImmediate call. |
</threat_model>
