  if (this.redis) {

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin WhatsApp message → knowledge base | The gate: only messages from verified admin (AdminIdentityService.canReceiveLearningReply=true) can reach this code path |
| Redis confirmation key → knowledgeService.save() | Atomic DEL (returns 1) prevents double-write on Baileys duplicate events |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-02-01 | Spoofing | Confirmation gate reply | mitigate | `canReceiveLearningReply` from `AdminIdentityService` is already checked upstream before `processAdminReply` is called — non-admin messages never reach this method |
| T-8-02-02 | Tampering | Double ingestion via Baileys duplicate events | mitigate | `redis.del(confirmationKey)` returns the count of deleted keys; only proceed if `deleted === 1` — second Baileys event gets `deleted === 0` and exits without calling `knowledgeService.save()` |
| T-8-02-03 | Repudiation | Knowledge ingested without admin awareness | mitigate | Echo message sent before ingestion; `taughtBy` field stores `adminJid` for audit trail (APR-05 extends this in Plan 03) |
| T-8-02-04 | Information Disclosure | Confirmation key leaks pending answer | accept | Key TTL=600s and stored in Redis (not DB); no PII beyond the admin's own answer |
</threat_model>
